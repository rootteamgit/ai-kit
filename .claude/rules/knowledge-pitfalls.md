# knowledge/pitfalls ルール

## 目的

AI が実装で犯したミスを記録し、同じミスの再発を防ぐ
溜まったらプロジェクトのコーディング規約（`.claude/rules/`）に昇格させる

## ファイル

`knowledge/pitfalls.jsonl` にフラットに記録する。フィールドの詳細は `knowledge.md` 参照

## tags のルール

「どういう実装をするときにこのミスを踏むか」で付ける
技術要素・ライブラリ名・機能カテゴリを kebab-case で指定する

例: `prisma,db` / `auth,oauth` / `react,hooks`

## 誰が書くか

実装AI（impl-ai）が書く。修正した内容のうち再発しそうなものを記録する

## 何を書くか

- 別のタスクで同じミスを繰り返しそうなもの（「このライブラリの API を間違えて使った」）
- ミスの原因がコードからは分からないもの（「この設定値は公式ドキュメントに書いてない」）
- 原因と正しい方法をセットで書く

## 何を書かないか

- タイポ、単純な構文エラー
- そのタスク固有で再発しないもの
- 一時的なエラー（ネットワーク障害、レート制限）

## 判定基準（再発ルール）

別のタスクで **同じミスを繰り返しそうか？**
Yes → 書く、No → 書かない
