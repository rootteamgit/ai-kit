# テストルール

## 方針

- 純粋関数には必ず Vitest ユニットテストを書く

## ファイル

- ソースと同じディレクトリに `*.test.ts` で配置

## テストの書き方

- describe でテスト対象（関数名・クラス名）をグループ化する
- 正常系・異常系・境界値の3パターンを書く
- 1テスト = 1つの振る舞い
- テスト名は「条件 → 結果」（例: '空文字はエラーを返す'）
- 各テストで独立したデータを用意する
- 非同期の rejects/resolves は必ず await する

## 既存テストの維持

- 実装後に `npm run test` で既存テストが通ることを確認する

## やらないこと

- スナップショットテストは使わない
