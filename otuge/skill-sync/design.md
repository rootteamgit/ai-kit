# スキル同期

## Overview

- ai-kit リポジトリを `.claude/skills/` と `.claude/tools/` のマスターとし、各プロジェクトに配布する仕組み
- 開発者が手動でCLIを実行すると、GitHub から最新のスキル・ツールを取得し配置する
- 各プロジェクトの `.claude/skills/` と `.claude/tools/` は gitignore し、ローカルで自由に改変できる

## 関連

なし

## 仕組み

### 全体フロー

1. 開発者がプロジェクトのルートで同期コマンドを実行する
2. `gh` コマンドで ai-kit リポジトリのファイル一覧を取得する
3. `.claude/skills/` と `.claude/tools/` 配下のファイルを取得し、実行先プロジェクトにコピーする
4. 衝突処理のルールに従い、差分やローカル固有ファイルを報告する

### 同期スクリプト

- Bash スクリプトで実装する
- ai-kit の `scripts/claude-sync.sh` に配置する
- 各プロジェクトで `gh` を使ってこのスクリプトを取得し、`bash` で実行する
- `gh api repos/{owner}/{repo}/git/trees/main?recursive=1` でファイル一覧（パス・SHA）を取得
- 各ファイルの内容は `gh api repos/{owner}/{repo}/git/blobs/{sha}` で取得（base64 デコードして書き出す）
- owner/repo とブランチ(main)はスクリプト内に埋め込む
- `.claude/skills/` と `.claude/tools/` を対象とする
- サブディレクトリも再帰的に処理する

### 同期ルール

- ai-kit にあってローカルにない → コピー
- ai-kit にあってローカルにもある → 差分チェックし、差分があればユーザーに上書き/スキップを質問する
- ai-kit になくてローカルにある → 「ai-kit に無いファイル」として報告する

### 各プロジェクト側の準備

- `.gitignore` に `.claude/skills/` と `.claude/tools/` を追加する
- `gh` コマンドが認証済みであること（未認証の場合はエラーメッセージを出して終了する）

### プロジェクト固有の情報

- スキル・ツール内にプロジェクト固有の値（ブランチ名等）をハードコードしない
- 汎用的な表現で記述する（例: `develop...HEAD` → 「開発ブランチとの差分」）
- Claude が CLAUDE.md やリポジトリの状態から具体的な値を判断する

### 成長フロー

1. ai-kit でスキル・ツールを管理・配布する（マスター）
2. 各プロジェクトではローカルで自由に改変・実験する（gitignore されている）
3. 改善が十分に育ったら、開発者が手動で ai-kit に還元する
4. 次回の同期で全プロジェクトに反映される

## やらないこと

- プロジェクト固有スキルの管理（必要になった時に対応）
- テンプレート変数の置換（まず汎用表現で運用）

## Decision Log

### 2026-04-09: スキルの一元管理方式

- 背景: スキルが各リポジトリに分散しており、更新・管理が煩雑
- 候補:
  1. 同期方式 — ai-kit がマスター。配布は自動、集約は手動
  2. 集約方式 — 各リポジトリがマスター。集約は半自動、配布の概念なし
- 決定: 同期方式
- 理由: マスターが明確で管理しやすい。集約方式だとどのリポジトリのスキルが最新か不明になる

### 2026-04-09: 取得方法に gh コマンドを採用

- 背景: GitHub からスキルファイルを取得する方法の選定
- 候補:
  1. GitHub API — 認証を自前で実装する必要がある
  2. `raw.githubusercontent.com` — ディレクトリ一覧が取得できない
  3. `gh` コマンド — 認証済み、プライベートリポジトリも対応
- 決定: `gh` コマンド
- 理由: 認証が既に通っている。プライベートリポジトリでもそのまま動く

### 2026-04-09: プロジェクト固有情報の扱い

- 背景: スキル内のブランチ名等がプロジェクトによって異なる
- 候補:
  1. テンプレート変数（`$#{base_branch}` 等）で同期時に置換
  2. スキルを汎用的な表現で書き、Claude に判断させる
  3. CLAUDE.md に値を定義し、スキルから参照させる
- 決定: まず汎用表現で書く。無理が出たら `$#{...}` を導入
- 理由: テンプレート変数の仕組みが不要になり、同期スクリプトがただのコピーで済む

### 2026-04-09: プロジェクト固有スキルの扱い

- 背景: 特定プロジェクトでしか使わないスキルの管理方法
- 決定: 今は対応しない
- 理由: 実際にどれくらい発生するか不明。発生してから考える
