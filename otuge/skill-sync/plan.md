# 実装プラン: スキル同期

## ファイル構成

```
ai-kit/
└── scripts/
    └── claude-sync.sh   # 同期スクリプト本体
```

## 実装順序とやること

### Step 1: scripts/ ディレクトリの確認・作成

- `scripts/` ディレクトリが存在しなければ作成する
- `claude-sync.sh` ファイルを新規作成する

### Step 2: スクリプトの骨格を実装

- shebang と基本設定 (`set -euo pipefail`)
- owner/repo とブランチ定数を埋め込む (`rootteamgit/ai-kit`, `main`)
- 対象ディレクトリ定数を定義 (`.claude/skills` と `.claude/tools`)

### Step 3: 前提条件チェックを実装

- `gh` コマンドの存在確認
- `gh auth status` で認証状態を確認し、未認証ならエラーメッセージを出して終了
- `base64` コマンドの存在確認（ファイル内容デコードに使用）
- `jq` の存在確認は不要（`gh api --jq` で代替するため）

### Step 4: ファイル一覧取得を実装

- `gh api repos/{owner}/{repo}/git/trees/main?recursive=1 --jq` でツリーを取得し、`.claude/skills/` または `.claude/tools/` に属するファイルのみフィルタ
- パスと SHA のペアを保持する

### Step 5: 同期ルールを実装

#### 5-1: ai-kit にあってローカルにない → コピー

- ファイルの親ディレクトリを `mkdir -p` で作成
- `gh api repos/{owner}/{repo}/git/blobs/{sha}` でファイル内容取得
- レスポンスの `content` フィールドを base64 デコードして書き出す

#### 5-2: ai-kit にあってローカルにもある → 差分チェック

- ローカルファイルの内容と取得内容を比較する
- 差分がなければスキップ（メッセージ不要）
- 差分があればユーザーに上書き/スキップを質問する (`read -r`)
- 上書きを選択したらファイルを書き出す

#### 5-3: ai-kit になくてローカルにある → 報告

- 同期完了後、ai-kit に存在しないローカルファイルを一覧表示する

### Step 6: 実行結果サマリーの表示

- コピーしたファイル数、上書きしたファイル数、スキップ数、ローカル固有ファイル数を表示する

## 実装上の注意

- `jq` は使わず `gh api --jq` で JSON を処理する
- base64 デコードはプラットフォームによりオプションが異なる (`-d` vs `-D`)。Linux/Mac両対応にする
- API レスポンスのエラーハンドリング（rate limit 等）は基本的なものにとどめる（design に記載なし）
- インタラクティブな質問 (`read`) は TTY がある場合のみ機能する前提で実装する
